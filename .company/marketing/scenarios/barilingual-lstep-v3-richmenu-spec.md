## 運用アクション5種

### 1. 運用_見積送付完了
- トリガー: スタッフが見積を送付した後に手動実行
- 処理:
  - 記録_見積送付日時 = アクション実行日時
  - 記録_現在フェーズ = 06
  - メニュー_03_見積比較訴求 に切替
  - シナリオ_03_見積比較訴求 開始

### 2. 運用_面談参加記録
- トリガー: 面談実施後にスタッフが手動実行
- 処理:
  - 記録_面談参加日時 = アクション実行日時
  - 記録_現在フェーズ = 08
  - メニュー_04_申込後押し に切替
  - シナリオ_04_申込後押し 開始

### 3. 運用_入金待ち移行
- トリガー: 申込受付後にスタッフが手動実行
- 処理:
  - 記録_入金期限 = 手動入力
  - 記録_現在フェーズ = 09
  - リマインダ_04_入金催促 開始

### 4. 運用_入金確認
- トリガー: 入金確認後にスタッフが手動実行
- 処理:
  - 記録_入金日時 = アクション実行日時
  - 記録_現在フェーズ = 10
  - メニュー_06_渡航準備 に切替
  - シナリオ_06_渡航準備 開始

### 5. 運用_休眠移行
- トリガー: 一定期間反応なし、またはスタッフ判断で手動実行
- 処理:
  - 記録_休眠移行理由 = 手動入力
  - 記録_現在フェーズ = 99
  - メニュー_05_再興味喚起 に切替
  - シナリオ_05_再興味喚起 開始

## 実装ファイル

| ファイル | 説明 |
|----------|------|
| src/actions/create_richmenu.js | リッチメニュー作成アクション（汎用） |
| src/v3-richmenu-runner.mjs | リッチメニュー6種一括作成ランナー |

## 実行手順

```bash
# 1. リッチメニュー6種を作成
node src/v3-richmenu-runner.mjs

# 2. 運用アクション5種を作成
node src/v3-actions-runner.mjs
```

注意: 全てOFF状態で作成。本番反映はLstep管理画面から手動で行う。
