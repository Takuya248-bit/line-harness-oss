# IG Auto Poster v2 - バリ島情報カルーセル自動投稿

## 概要

バリリンガルのInstagramカルーセル投稿を、英語学習ネタからバリ島ローカル情報（カフェ/スポット/グルメ等）に全面転換する。写真背景+テキストオーバーレイのデザインで、ネタ選定→情報調査→写真取得→スライド生成→承認→投稿を全自動化する。

## 背景

- 現状: 英語フレーズ学習カルーセル（Satori SVG生成、60ネタハードコード）
- 課題: 競合多い、差別化弱い、テンプレ感が強い、エンゲージメント低い
- 機会: バリ島情報系はIG上で競合ゼロ、実際のバリリンガルの投稿でもバリ情報系がエンゲージメント高い
- 調査: カルーセルはエンゲージメント率10%超（リール6%、単一画像7%を上回る）。保存が2026年の最重要指標

## コンテンツ戦略

### ネタカテゴリ（ローテーション）

エリア×ジャンル型:
- エリア: チャングー/ウブド/スミニャック/サヌール/ヌサドゥア/クタ/ジンバラン/ウルワツ
- ジャンル: カフェ/レストラン/スパ/映えスポット/ビーチ/ショップ

テーマ型:
- ヌサペニダ日帰り/バリの朝活スポット/ローカルフード/ナイトスポット/寺院巡り/サーフスポット/親子で楽しめる場所

### 投稿構成（10枚カルーセル）

1. カバー: エリア写真全面 + キャッチコピー（例:「チャングーで行きたい！おしゃれカフェ5選」）+ Balilingualロゴ
2-6. 各スポット: 写真全面 + 番号バッジ(オレンジ丸) + 店名/スポット名 + 紹介文(3-4行) + 営業時間等
7. まとめ/MAP: 5スポットの位置関係やリスト
8. CTA: 「保存してバリ旅行の参考に」「詳しくはプロフィールのリンクから」

### キャプション構成

- 1行目: タイトル（カバーと同じ）
- 本文: 各スポットの一言紹介（リスト形式）
- CTA: 「保存して旅行の参考に」「友達にシェアしてね」
- ハッシュタグ: バリ島関連10個（#バリ島 #バリ旅行 #バリ島カフェ #バリ島観光 #バリ島留学 #チャングー #ウブド #バリリンガル 等、エリア/ジャンルに合わせて動的生成）

## デザイン仕様

### カバースライド
- 写真全面背景（1080x1350px、4:5比率）
- 上部: Balilingualロゴ（白、センター）
- 中央: キャッチコピー（白、太字、ドロップシャドウ）
- 番号バッジ:「5選」等（オレンジ丸）
- 下部余白に白枠線（スクショ参照）

### スポットスライド
- 写真全面背景
- 左上: 番号バッジ（オレンジ丸、NO.1〜5）
- 上部: 店名/スポット名（白、太字、大きめ）
- 下部: 紹介文（白、通常weight）+ 営業時間（時計アイコン）
- テキスト背景: 半透明黒グラデーション（下部30%）で視認性確保

### 共通
- フォント: Zen Maru Gothic（現行踏襲）
- テキスト: 白 + text-shadow(0 2px 8px rgba(0,0,0,0.7))
- アクセント: オレンジ(#E67E22)
- ロゴ: Barilingual（白、上部固定）

## 自動パイプライン

### 1. ネタ選定（Claude API）
- カテゴリプールからランダム選択（重複回避はD1で管理）
- エリア×ジャンルまたはテーマを決定
- 出力: テーマ、エリア、スポット候補5件

### 2. 情報調査（Claude API + Web検索）
- 各スポットの情報をWeb検索で調査
- 店名、エリア、特徴、営業時間、おすすめポイント
- 紹介文（3-4行、カジュアルで親しみやすいトーン）

### 3. 写真取得（Unsplash API）
- 各スポット名/エリア名で検索
- 4:5比率にcrop
- 無料枠: 50リクエスト/時間
- フォールバック: エリア名で汎用的なバリ島写真を取得

### 4. スライド生成（Satori + resvg-wasm）
- 写真をfetchしてSatoriのimg要素として背景配置
- テキストオーバーレイ（白+シャドウ）
- PNG変換は1枚ずつ（CPU制限対策）

### 5. 承認（Webギャラリー）
- /gallery: 生成済みコンテンツ一覧（ステータスフィルタ付き）
- /gallery/:id: 全スライドプレビュー + 承認/スキップボタン
- LINE通知は投稿完了時のみ

### 6. 投稿（Instagram Graph API）
- Cron Trigger: 1日2回（08:00/16:00 バリ時間）
- 承認済みコンテンツをカルーセル投稿

## DB設計（generated_content拡張）

既存のgenerated_contentテーブルを拡張:
- template_type → カテゴリ（cafe_canggu, spot_ubud, theme_nusapenida等）
- content_json → スポット情報、写真URL、紹介文等をJSON格納
- status: pending_review / approved / posted / skipped / rejected

投稿履歴テーブル（重複回避用）:
- posted_topics: テーマ+エリアの組み合わせを記録

## 技術的な注意点

- Unsplash API: 利用規約で帰属表示が必要。キャプションに「Photo by X on Unsplash」を追加
- Satoriでの画像表示: img要素のsrcにURLを指定すればfetchして埋め込む（width:100%, height:100%, objectFit:cover）
- CPU制限: 写真入りのPNG変換は現行より重くなる可能性。Workers有料プラン($5/月)が必要になる可能性あり
- IG Access Token: 60日で失効する長期トークンの場合、定期更新の仕組みが必要

## 既存コード（英語学習テンプレート）の扱い

- content-data.ts（60ネタ）: 削除せず残す。フォールバックとして機能
- 既存テンプレート（7種）: 当面残す。バリ情報テンプレートが安定したら削除検討
- 新テンプレート: templates/bali-cover.ts, templates/bali-spot.ts, templates/bali-cta.ts を新規作成

## 成功指標

- 保存数: 投稿あたり平均50保存以上（現状の英語学習投稿との比較）
- エンゲージメント率: 10%以上
- フォロワー増加: 月+200人ペース
