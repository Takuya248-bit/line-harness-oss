# VideoTune(仮) - AI動画編集Botサービス設計

## 概要

動画を送るだけで、使うほど自分のチャンネルに最適化されるAI編集アシスタント。

## ターゲット

Vlog・解説動画・ショート動画を作る一般クリエイター（料理に限らない）。

## MVP機能

1. SE自動挿入（cooking-sfx-botから移植・汎用化）
2. ジェットカット（無音/無意味な間の自動カット）
3. 学習エンジン（SE配置 + ジェットカットのテンポ感をユーザー単位で学習）

次フェーズ: カット割（既存動画からスタイル学習 + ジャンル別プリセット）

## コア差別化

「使うほど自分のチャンネルに最適化される」学習機能。競合（Vrew, CapCut, Descript）にはない。LINEの手軽さ、オールインワン、安さは支える要素。

## アーキテクチャ

```
video-editor-bot/（新プロジェクト）
├── core/                  # 共通処理エンジン
│   ├── sfx/              # SE挿入（cooking-sfx-botから移植・汎用化）
│   ├── jet_cut/          # ジェットカット（新規）
│   └── learning/         # 学習エンジン（現行learning.pyを拡張）
├── bot/                   # LINE Bot（FastAPI）
├── sessions/             # ユーザーセッション管理
└── tests/
```

- cooking-sfx-botは櫻子さん専用として維持、壊さない
- core/を分離しておくことで、将来Webダッシュボードやapi提供時にそのまま使える
- 学習エンジンはSE・ジェットカット両方に共通の仕組み（操作記録→ルール導出）

## ジェットカット機能

### 処理フロー

1. 動画の音声トラックを抽出（ffmpeg）
2. 音声レベル解析 → 無音区間を検出（閾値ベース）
3. 無音区間をカット → 前後の映像をつなぐ
4. トランジション処理（ハードカット or クロスフェード）

### 学習が効くポイント

- 無音閾値（チャンネルによって「間」の取り方が違う）
- 最小残し時間（完全に無音をゼロにすると不自然。0.1〜0.3秒の調整）
- カット後のテンポ感（1分あたり何カットが心地よいか）

### ユーザーの調整方法

- 「もっとテンポよく」「間をもう少し残して」等のテキスト指示
- SE同様、調整操作を学習データに蓄積

### 初期プリセット（ジャンル別デフォルト値）

- ゆったり系（料理、Vlog）: 閾値低め、間を多めに残す
- テンポ系（解説、レビュー）: 閾値高め、タイトにカット
- ショート動画: 最もアグレッシブにカット

## 学習エンジン

### 構造

```
学習エンジン（共通基盤）
├── 記録層: 操作ログの永続化（何を、どう調整したか）
├── 導出層: パターン → ルール変換（閾値到達で自動ルール化）
└── 適用層: パイプライン実行時にルールを注入
```

### 学習データ（ユーザー単位）

- `{user_id}/adjustments.json` - 全操作ログ
- `{user_id}/rules.json` - 導出されたルール
- `{user_id}/profile.json` - ジャンル、好みのテンポ等（初回選択 or 自動推定）

### 学習サイクル

1. 初回: プリセット（ジャンル選択）で処理
2. ユーザーが調整指示を送る
3. 操作を記録 + ルール導出
4. 次回の動画から自動適用
5. 繰り返すほど調整不要に近づく

## LINE Botユーザーフロー

### 初回

```
友だち追加 → 「ようこそ！ジャンルを選んでください」
→ [料理/Vlog/解説/レビュー/ショート/その他]
→ プリセット設定完了、1週間無料トライアル開始
```

### 通常利用

```
動画を送信
→ 「処理中...（30-60秒）」
→ SE挿入 + ジェットカット済み動画のDLリンク
→ タイムライン表示（SE配置 + カットポイント）
```

### 調整

```
「もっとテンポよく」「5秒のSE消して」「間をもう少し残して」
→ 再生成 → DLリンク（学習データに記録）
```

### コマンド

- 「SE一覧」「学習状況」「設定変更」「使い方」

## 技術スタック・インフラ

- サーバー: Fly.io（自動スケール、GPU不要）
- 動画一時保存: Fly Volume（処理後24時間で自動削除）
- 学習データ永続化: Cloudflare R2（S3互換、安価）
- 決済: Stripe（サブスク管理、期間プラン対応）
- 決済UI: LIFF（LINE内ブラウザでStripe Checkoutを開く）
- 監視: Fly.ioメトリクス + 処理キュー溢れアラート

## フェーズ計画

- Phase 1 (MVP): SE自動挿入 + ジェットカット + 学習エンジン + LINE Bot + Stripe決済
- Phase 2: カット割（既存動画からスタイル学習 + ジャンル別プリセット）
- Phase 3: Webダッシュボード（設定管理、履歴、分析）
- Phase 4: API提供（開発者・編集ツール向け）
