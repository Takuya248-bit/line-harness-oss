# ABテスト全シナリオ実装設計

## 概要
既存ABシナリオ（Day0-4）をベースに、v3の全シナリオにABテスト（メッセージ文面テスト）を導入する。

## 方針
- 既存ABシナリオ（ID: 1182229）は削除せず、そのまま稼働継続
- v3シナリオ（S_01〜S_05, R_01）の各CTA最終メッセージにA/B分岐を追加
- ABの振り分けは既存タグ `3/22_AB_A` / `3/22_AB_B` を再利用
- Aは現行文面をそのまま使用。Bを新規作成

## 実装範囲

### Phase 1: 既存ABシナリオの継続利用
- 既存ABシナリオ（Day0-4）はそのまま稼働 → S_02の代わりに使用
- テキスト短縮 + CTAバナーのパック化は行わない（現行パックがすでにその形式）
- 変更なし

### Phase 2: v3シナリオへのA/B分岐追加
対象6シナリオの最終CTA or CVに最も近いメッセージにA/B分岐を追加:

| シナリオ | 対象メッセージ | A（現行） | B（新規作成） |
|---------|-------------|----------|-------------|
| S_01 | Day5 最終リマインド | 現行パック v3_01_Day5_最終リマインド | v3_01_Day5B_最終リマインド |
| S_03 | Day7 最終フォロー | 現行パック v3_03_Day7_最終フォロー | v3_03_Day7B_最終フォロー |
| S_04 | Day5 申込後押し | 現行パック v3_04_Day5_申込後押し | v3_04_Day5B_申込後押し |
| S_05 | Day56 キャンペーン | 現行パック v3_05_Day56_キャンペーン | v3_05_Day56B_キャンペーン |
| R_01 | Day5 最終 | 現行パック v3_R01_5日後_最終 | v3_R01_5日後B_最終 |

S_02はなし（既存ABシナリオが担当）
S_06, R_02〜R_04はなし（入金済み/運用系でテスト不要）

### 実装手順（各シナリオ共通）

1. B用テンプレートパック（テキスト+CTAバナー）を作成
2. 既存シナリオの対象メッセージの絞り込み条件に `タグ[3/22_AB_A]あり` を追加
3. 同じDay・同じ時刻にB用メッセージを挿入し、絞り込み条件 `タグ[3/22_AB_B]あり` を設定
4. B用メッセージにも既存と同じアクション（フェーズ変更等）を紐付け

### 配信条件チェックリスト（実装後の必須確認）
各A/Bメッセージペアに対して:
1. A用: 絞り込み = `3/22_AB_A` タグあり ✓
2. B用: 絞り込み = `3/22_AB_B` タグあり ✓
3. 配信日数・時刻が完全一致 ✓
4. アクション設定が同一（フェーズ変更等） ✓
5. タグなし(A/Bどちらでもない)の人に配信されない ✓

- 各シナリオのCTA選択率をA/B別に比較
