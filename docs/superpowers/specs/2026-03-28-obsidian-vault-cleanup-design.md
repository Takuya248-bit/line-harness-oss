# Obsidian Vault ファイル整理 設計書

## 概要

1,162ファイルのObsidian Vaultを整理する。プロジェクト中心のフォルダ構成に再編し、重複フォルダを統合、00_inboxの495件をLLM自動分類して振り分ける。

## 課題

- 構造の混乱: 重複フォルダ（アイデア×2、知識×2、inbox×2）
- ノイズ: 全体の87%がYouTubeとinboxの未整理蓄積
- Vault全体の見通しが悪い

## 整理後のフォルダ構成

```
Obsidian Vault/
├── 櫻子/
│   ├── YouTube/          ← 既存516件（Processed含む）
│   ├── 企画/             ← 台本ルール、節の定義、分析
│   └── meetings/         ← 必要に応じて
├── バリリンガル/
│   ├── コース/           ← ヨガ留学、21日コース等
│   └── マーケティング/
├── Lカスタム/
│   └── 開発/             ← 既存「開発・実装」から移動
├── AI副業コース/          ← 既存のまま（8件）
├── knowledge/            ← 既存knowledge + 「知識」を統合
├── ideas/                ← 「アイデア」+「アイディアメモ」を統合
├── meetings/             ← プロジェクト横断の会議（出資者会議等）
├── templates/            ← 既存テンプレート
├── リードマグネット/      ← 既存のまま（3件）
└── archive/              ← inbox分類後の低価値ファイル退避先
```

## 00_inbox自動分類（495件）

### 対象

- 00_inbox/inboxchatgpt: 87件
- 00_inbox/inboxgemi: 212件
- 00_inbox/Perplexity: 196件

### 分類フロー

1. 各ファイルの内容を読み取り
2. LLM（Claude）でカテゴリ判定 + 価値スコア（1-5）を付与
3. スコア3以上 → 適切なフォルダに移動
4. スコア2以下 → archive/に移動

### 分類先カテゴリ

| カテゴリ | 振り分け先 |
|---------|-----------|
| 櫻子チャンネル関連 | 櫻子/ |
| バリリンガル関連 | バリリンガル/ |
| Lカスタム・開発関連 | Lカスタム/ |
| AI副業関連 | AI副業コース/ |
| 汎用ナレッジ | knowledge/ |
| アイデア・企画 | ideas/ |
| 分類不能・低価値 | archive/ |

### 実行方法

- サブエージェントで並行処理（バッチ50件ずつ）
- 分類結果をCSVで出力し、ユーザー確認後に移動実行
- `[[]]` 内部リンクがあれば移動後にパス更新

## 実行順序

1. git commitで現状スナップショット保存
2. フォルダ作成（櫻子/企画、バリリンガル/コース等）
3. 重複フォルダ統合（36件）
4. YouTube/を櫻子/YouTube/に移動（516件）
5. ミーティング文字起こし38件をLLM分類→確認→振り分け
6. 00_inbox 495件をLLM分類→CSV出力→確認→振り分け
7. 空フォルダ削除
8. git commitで整理後の状態を保存

## リスク対策

- 全操作前にgit commitで復元ポイント作成
- ファイル移動はmv（Obsidianキャッシュ整合性維持）
- `[[]]` 内部リンクは移動後にパス更新
- 万一のミスはgit restoreで復元
