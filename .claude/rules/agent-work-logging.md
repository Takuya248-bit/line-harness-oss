# エージェント作業ログ（全作業者必須）

全てのエージェント（メイン・サブ問わず）は、作業完了時に必ず `.company/secretary/notes/YYYY-MM-DD-progress.md` に追記すること。

## ログ記載タイミング

- 作業完了時（成功・失敗問わず）
- コミット・push実行後
- エラー発生時（原因と対処を含める）

## ログ形式（厳守）

```
## HH:MM [作業者名/エージェント種別]
- 作業内容: （何をしたか）
- 対象: （ファイルパス、サービス名等）
- 結果: 完了 / 部分完了 / 失敗
- 変更点: （主な変更の箇条書き）
- 備考: （次のアクション、注意点等。なければ省略）
```

## 必須ルール

1. 同日ファイルが存在する場合は末尾に追記（上書き厳禁）
2. コミットした場合はコミットハッシュを記載
3. ログを残さない作業は「やっていない」と同じ。例外なし
4. 日付は `date +%Y-%m-%d` で確認してから書き込む
5. 時刻は `date +%H:%M` で取得する

## サブエージェントの注意

- 返答をメインエージェントに返す前にログを書くこと
- ログ書き込みに失敗した場合、返答にその旨を含めること
- 調査のみ（ファイル変更なし）でも、調査結果をログに残すこと
