# 週次セルフレビュー

毎週の振り返りで失敗パターンを検出し、ルール改善に繋げる。

## タイミング

- 毎週月曜の /ohayou 実行時、または週初のセッション開始時に秘書が提案
- 「週次レビュー」「振り返り」とユーザーが言った場合も実行

## レビュー手順

1. 過去7日分の progress.md を集計
2. 過去7日分の decisions.md を確認
3. 以下のフォーマットで `secretary/notes/YYYY-WXX-review.md` を作成

## レビューフォーマット

```
# 週次レビュー YYYY-WXX（MM/DD〜MM/DD）

## 今週の実績
- 完了タスク数: X件
- 失敗/リトライ: Y件
- 新規ルール追加: Z件

## 繰り返しパターン（要注意）
- （同じミスや同じ指摘が複数回あったもの）

## フィードバック昇格候補
- （2回以上出たフィードバックでまだルール化されていないもの）

## エージェント品質トレンド
- リサーチ: 改善 / 横ばい / 悪化
- コンテンツ: 改善 / 横ばい / 悪化
- 実装: 改善 / 横ばい / 悪化

## 来週の改善アクション
- （具体的に1〜3個）
```

## ルール

- レビューファイルは週1つ（同一週は追記）
- 品質トレンドの判断根拠を1行添える
- 改善アクションは実行可能な粒度にする（「気をつける」は禁止）
