# フィードバック昇格ルール

同じフィードバックを2度与えない仕組み。修正の複利効果を最大化する。

## 昇格フロー

1. 初回指摘: memory/ にfeedback_*.mdとして保存（現行通り）
2. 同種の指摘が2回目に発生: 秘書が `.claude/rules/` へのルール昇格を提案
3. ユーザー承認後: ルールファイルを作成し、memory/の元フィードバックに「昇格済み」と追記

## 秘書の責務

- ユーザーからの指摘時、memory/の既存フィードバックと照合する
- 類似フィードバックが存在する場合:「以前も同じ指摘がありました。ルール化しましょうか？」と提案
- 昇格したルールは全エージェントに自動適用される（.claude/rules/は全員が読む）

## ルール化の基準

- 昇格する: 2回以上の同種指摘、プロジェクト横断で適用可能、再発防止効果が高い
- 昇格しない: 1回限りの状況依存、個人の好み（memoryのまま）、既にルール化済み

## 定期棚卸し（週次レビュー時に実施）

- memory/のfeedback_*.md を確認
- 既にルール化済みのフィードバック → memoryに「昇格済み→rules/xxx.md」と追記
- 3週間以上参照されていないフィードバック → 削除候補としてユーザーに提示
