# コンテキスト管理ルール

全エージェントはコンテキスト消費を意識し、以下の予算内で動作する。

## ファイル読み込み予算

| 種別 | 上限 | 備考 |
|------|------|------|
| スキルインデックス | 100行 | SKILL.md本体。詳細はreferences/に分離 |
| 1ステップで読む参照 | 200行 | 必要な参照ファイル1-2個分 |
| リサーチ結果の合成 | 200行 | サブエージェントからの結論のみ |
| CLAUDE.md | 60行以内推奨 | 詳細ルールは.claude/rules/に分離 |

## 同時ロード制限

- 参照ファイル同時ロード: 2ファイルまで
- 3+ファイル同時ロードは設計の警告サイン → 分割を検討
- 大きなファイルはoffset/limitで必要部分だけ読む

## サブエージェント返答上限

| 役割 | 上限 | 根拠 |
|------|------|------|
| リサーチ系 | 1000文字 | 調査結果は情報量が必要 |
| コンテンツ系 | 1000文字 | 表現の質を確保 |
| 実装系 | 500文字 | 結果とコミットハッシュ中心 |
| 検証系 | 300文字 | 判定+根拠+指摘のみ |

- メインコンテキストに戻すのは「結論と根拠」のみ
- 調査過程の詳細、中間結果、生データはサブエージェント内で完結させる

## セッション開始時の調査手順（必須）

新タスクに着手する前に、以下の順序で情報を集める。上で十分なら下に進まない。

1. progress.md / decisions.md を読む（引き継ぎ情報・リポジトリパス・完了済み作業）
2. 設計ドキュメント（docs/specs/, docs/plans/）があればそれを読む
3. 上記で判明したファイルパスに直接 `ls` + `Read` でアクセス
4. それでも不足する場合のみ Explore エージェントを1回起動

禁止:
- progress.mdに書いてある情報を無視してExploreで再調査する
- 「全体把握」目的でExploreを複数回起動する（1回で足りなければ的を絞ってGlob/Grep）
- 設計ドキュメントがあるのにコード全体を読み直す

## トークン節約の優先順位

1. 既存の引き継ぎ情報（progress.md）を最初に読む
2. 不要なファイル読み込みを避ける（offset/limit指定）
3. サブエージェントに委譲して結論のみ受け取る
4. スキルの段階的開示（references/分離）
5. CLAUDE.mdの簡潔化（詳細はrules/に分離）
6. 閾値で/compact提案（エージェント8回 or 往復20回）。1セッション=1プロジェクトを原則とし、プロジェクト切替時はcompactではなくセッション切替を推奨

## 予算超過時の対応

- 1ファイルが500行を超える場合: 分割を検討
- スキルが500行を超える場合: references/に段階的開示化
  - SKILL.md本体: ナビゲーション+ステップ概要のみ
  - references/: 各ステップの詳細手順
  - 現在のステップに必要な参照だけ読み込む
- CLAUDE.mdが100行を超える場合: rules/への分離を検討
