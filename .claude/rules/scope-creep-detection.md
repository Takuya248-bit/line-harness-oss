# スコープクリープ検出ルール

タスク分解時にスコープの膨張を検出し、早期に警告する。

## 自動警告トリガー

1. サブタスク数が15を超えた場合
   → 「サブタスクが15件を超えています。スコープが膨張していませんか？」と警告
   → フェーズ分割を提案する

2. 元の要件に遡れないサブタスクが存在する場合
   → 「このタスクの出自が不明です。元の要件のどれに対応しますか？」と確認
   → 対応する要件がなければ削除候補

## 出自追跡（必須）

タスク分解時、各サブタスクに出自を1行添える:
```
### サブタスク名
- 出自: （元の要件/ユーザー指示/調査結果のどれに基づくか）
- Done when: ...
```

## スコープ判定基準

- 必須: ユーザーが明示的に依頼したもの
- 推奨: 依頼を満たすために技術的に必要なもの
- 削除候補: 「ついでにやっておいた方がいい」レベルのもの

削除候補はユーザーに提示して判断を仰ぐ。勝手に含めない。
