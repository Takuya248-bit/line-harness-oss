# セッション規律ルール

## 1. セッションスコープ宣言（必須）

セッション開始時（/ohayou後）にユーザーが最初の指示を出したら、秘書が以下を宣言する:

```
[セッションスコープ] 今回: {プロジェクト名}
```

1セッション = 1プロジェクト（1リポジトリ）を原則とする。
別プロジェクトに着手したくなったら「プロジェクト切替しますか？」と確認する。

## 2. ブロッカー即記録（必須）

ブロッカーが発覚した時点で progress.md に記録する。TODOファイルとの二重管理はしない（progress.mdが正）。

形式: `- [BLOCKER] {内容} | 優先度: 高 | 発覚: YYYY-MM-DD`

## 3. 意思決定の即時記録（必須）

意思決定が発生したら、その場で decisions.md に記録する。
「後で書く」「未記録」は禁止。決定した瞬間に書く。

秘書のチェック: compact前スナップショットで「decisions.md未記録」が1件でもあれば警告。

## 4. セッション終了時クリーンアップ（必須）

ユーザーが /clear や /compact を実行する前、または「終わり」「おつかれ」と言った場合:

1. `git status` で未コミット変更を確認
2. 未コミット変更がある場合:
   - 意図的な変更 → コミット+push
   - 不要な変更 → ユーザーに確認して破棄
   - 他プロジェクトの変更 → 「{ファイル}が未コミットです」と警告
3. TODOファイルの更新（完了分チェック、新規追加）

## 5. compact回数の自己監視

1セッション内でcompactが3回を超えたら:
- 「compactが{N}回目です。セッションを分割しましょうか？」と提案
- スコープが広がりすぎている兆候
