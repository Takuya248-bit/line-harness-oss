現在のブランチのPR差分をレビューする。

手順:
1. `git diff main...HEAD` で変更差分を取得
2. 差分がなければ `git diff HEAD~1` で直近コミットの差分を取得
3. 以下の観点でレビューを実施:

セキュリティ:
- APIキーや秘密情報のハードコードがないか
- SQLインジェクションの脆弱性がないか（.bind()を使っているか）
- wrangler.tomlにdatabase_idが含まれていないか
- eval + curlなどシェルインジェクションのリスクがないか

コード品質:
- バグやロジックエラーがないか
- TypeScriptの型安全性が確保されているか
- エラーハンドリングが適切か

パフォーマンス:
- 不要なN+1クエリがないか

4. 以下のフォーマットで報告:

## レビュー結果
### 総合評価（LGTM / 要修正 / 要確認）
### セキュリティ（問題点または「問題なし」）
### コード品質（指摘事項または「問題なし」）
### 改善提案（あれば）
